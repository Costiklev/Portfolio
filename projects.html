My first real game, rough code, janky graphics, but surprisingly fun to play.

This is the first real game I ever made, everything before it was just broken prototypes used to figure out how coding even works. Built in GameMaker using GML, it’s a simple arcade-style project where the player moves around the screen, dodging enemy blocks and trying to survive as long as possible.

The controls are simple: use WASD to move around and Ctrl to speed up. But the fun comes from the obstacles. They appear in waves, starting with basic blocks that fall straight down, then ramp up with enemies that have custom movement patterns, unique shapes, and chaotic behavior.

There are also a couple of leftover debug controls: press Space to skip to the next level section, and use the mouse scroll wheel to change the player's size. I originally added these for testing, and planned to remove them, but never got around to it.

The game has no menus (it used to, but some files got corrupted along the way), and everything happens in a single scene.

All the textures were made in Paint 3D, a 3D modeling tool that I somehow used as a 2D art program. The result? Visuals that are as simple (and ugly) as they come.

I made this project roughly 5–6 years ago, and it took me about a month to finish. Even with all its flaws, I look back on it as my first complete game, and honestly, I still find the gameplay kind of addicting.

A fast-paced, space-themed, arcade-style 2D runner.

Lunar Dash is my first official game project. It's a fast-paced, space-themed 2D runner that I began working on in 2021 and after about a year of development, I released it on the Google Play Store.

Inspired by the simplicity and addictiveness of Flappy Bird, the game let's you take control of an astronaut dashing across the moon with the goal of surviving as long as possible while dodging deadly obstacles and collecting coins. You have two core abilities: a variable-height jump and a mid-air dash, both designed to give players tight control over movement and timing.

The initial release (v1.0) featured a single infinite game mode and just one type of obstacle. But development didn’t stop there.

In the upcoming v2.1.1 update (currently unreleased on the Play Store), I added a range of new features:

· New obstacles: acid puddles, alternate pipe shapes, and other hazards.

· Daily Level Mode: a handcrafted level that changes every 24 hours, offering a fresh challenge to players each day.

· Glide mechanic: by holding the screen while airborne, players can now slow their fall and gain better control mid-jump.

To top it off, the game includes a public leaderboard where players can submit their high scores from the infinite level and compete for the top spot. The leaderboard was built using a third-party online tool, integrated into the game to support real-time updates.

Daily Levels

Daily mode features 8 handcrafted levels, each with unique obstacles and designs. You'll find levels with huge acid puddles, bouncy mushrooms, horizontal pipes, and more.

Behind the scenes, a custom system assigns players one random level each day, the same for every instance, without needing any internet connection.

You get 3 lives to beat the level, shown in the top-left corner of the screen. Finish it successfully, and you'll earn 100 coins, which can be used for small advantages in infinite mode.

The menus:

In the development of Lunar Dash, I dove deep into UI/UX design to create smooth and intuitive user interfaces.

The game features six unique menus: the main menu, leaderboard, settings, pause, lose, and win screens.

Each one was carefully designed to be functional, responsive, and visually consistent with the overall game style. Through this process, I learned how to balance usability with aesthetics, ensuring the menus support the gameplay without getting in the way.

Behind the scenes:

Lunar Dash isn't just about fast gameplay and flashy colors, it's backed by custom systems and handcrafted logic to create a smooth and replayable experience.

In this section, I’ll showcase some of the actual code used in the game, along with thoughts on the logic we developers used to shape the gameplay experience. From movement mechanics to daily level generation, these scripts reflect both functionality and design intent — not just how the game works, but why it was built that way.

I developed Lunar Dash alongside two other teammates, taking on the role of project manager, lead programmer, and game designer.

Movement Code Snippet

This code snippet comes from the astronaut's movement script. It showcases how I implemented a variable jump height mechanic, designed to make both small and large jumps feel smooth and responsive, ensuring tight controls that match the game's fast-paced style.

I created this script with two main goals in mind: smooth responsiveness and cross-platform compatibility.

To make jumping feel natural and fluid, I didn’t just tweak the player’s Y velocity, instead, I controlled the gravity scale, allowing for both high and low jumps to feel smooth and intentional.

The second goal was making this system work seamlessly across all platforms. I built the input system to handle both button-based input (used by keyboard and controllers) and touch input (used on mobile devices, including Android and iOS). That way, the same logic could power the movement regardless of the device.

Daily Level Manager Script

This script handles the daily level system in the game. Its purpose is to pick a new level every 24 hours, giving players something fresh to play each day.

It works by creating a list of levels, each assigned an index starting from 0. Then, it generates a seemingly random number by multiplying the current day and month. That number is then reduced (by subtracting the total number of available levels repeatedly) until it fits within the valid range of indexes. Finally, the level corresponding to that index is instantiated and used as the daily level.

This approach guarantees that all players in the same time zone receive the same level, enabling synchronized challenges and allowing friends to compete fairly.

A relaxing, Suika Game inspired, puzzle game

Donut Fusion Frenzy is a casual, puzzle game inspired by the viral hit Suika Game. It combines colorful 2D cartoon visuals with a deceptively simple gameplay loop: throw and merge donuts of the same size to create bigger ones and gain points.

The challenge lies in managing a limited space, once the stack of donuts overflows the play area, it’s game over. There are eight unique donut sizes, with the largest one not only giving the most points but also disappearing upon merging, allowing players to keep going and chase high scores endlessly.

While the physics-driven gameplay may seem chaotic or unintuitive at first, especially due to the donuts’ bouncy behavior, experienced players can learn to anticipate movement patterns and even use the unpredictable bounciness to their advantage.

The Levels:

The game features 14 levels: 10 standard ones unlocked through regular progression, one arena where players can compete for the first place, and 3 special levels unlocked by completing a difficult challenge. Each of the 10 normal levels has its own unique look, theme, post-processing effects, shaders, and mechanics.

The first level acts as a gentle introduction, while each subsequent level adds new twists. For example:

- Level 3 hides the list of upcoming donuts

- Level 5 adds cactus obstacles that donuts can stick to

- Level 9 introduces lightning and cloud donuts, where lightning clears space and clouds block your vision.

The Arena is similar to any other level, but it allows players to publish on the game's leaderboard their high score once they finish playing.

The 3 special levels take things even further:

- The Space Level is octagon-shaped instead of square and has gravity that shifts direction based on which wall you shoot toward. It even lets players explore outside the map, where they might find references to games like Hollow Knight.

- Time Frenzy gives players a ticking timer that can be slowed by merging donuts, adding pressure and urgency to the gameplay.

- Simulation Mode allows players to tweak the experience before playing—adjusting donut size, gravity strength, or the layout of the level itself.

These special levels unlock once the player achieves a high score of 12,000 on every normal level, which usually takes around 2 hours of gameplay.

The Menus:

Compared to Lunar Dash, this game introduces a much more refined and expansive UI, packed with menus, elements, and transitions that enhance the player experience.

From the main menu, players can access every section of the game:

- On the bottom left, the Settings button opens a panel with categories like video, audio, language, profile, cheats, tutorial, credits, and more

- On the bottom right, the Skins button opens a menu where players can unlock or purchase new appearances for their donuts.

- Above the main button, you'll find four additional buttons for less frequently used features, such as the arena and the achievements

- Below it, there's a mission scroll. When clicked, it shows a list of available missions along with the rewards for completing them.

- Finally, the main “Play” button uses a smooth background transition to elegantly reveal all available levels, adding a nice polish to the overall experience.

A small-scale prototype inspired by Conway's Game of Life, but with an elemental twist.

Unlike my other projects like Lunar Dash and Donut Fusion Frenzy, this one was smaller in scale and built primarily as a portfolio piece while also helping me learn more about 2D arrays and optimization.

It's a clone of the classic Cellular Automata concept, where cells on a grid evolve into fascinating patterns by following simple behavior rules, but with a twist. Instead of being simply 'dead' or 'alive', each cell represents an element or material (like fire, water, earth, or wind) each with its own behavior based on its surroundings.

These four screenshots showcase different aspects of the game. While one image showcases the visuals of the game after minutes of gamplay, the other screenshots highlight other aspects on the backend side. For example cell behaviours, and grid generation.

The game uses very simple visuals (many are Unity’s default textures) since the primary focus was on building optimized, modular systems.

From the main menu, players can select the grid size. The in-game UI features two side panels with options like enabling auto-update, clearing the grid, and choosing which element to place. The goal was to create an intuitive sandbox environment that allows players to experiment freely.

Each cell has logic based on its element, so fire will expand if it has grass adjacent to it, but will die if there's no grass in its surroundings, water will expand infitely extinguishing fire and making plants grow more efficiently, rocks are simple obstacles that block a cell's behavious ecc.

Cells are created through scriptable objects, meaning that everything can be setup in Unity without modifying any code. Thanks to Unity's simple UI it's easy to create a cell from scratch, deciding it's behaviour, colour, and type without worrying about any scripts breaking.
